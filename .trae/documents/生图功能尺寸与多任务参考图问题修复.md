# 生图功能：图生图缺少参考图指示修复

## 摘要

针对 `gpt-image-2`（非 codex）模型在图生图模式下，多张任务最终生成的图片与上传的参考图无关（退化为纯文生图）的问题进行修复。

**核心根因**：非 codex 模型走 ChatGPT 网页接口（`/backend-api/f/conversation`），`build_image_prompt` 只拼接 `size` 和 `quality`，**不区分文生图/图生图**。上游 ChatGPT 收到 multimodal 输入（图+文），但文本没有明确指示"请基于参考图编辑"，模型可能直接根据 prompt 生成新图，忽略参考图。

对比 codex 模型（[openai_backend_api.py:756](file:///d:/GithubProject/chatgpt2api/services/openai_backend_api.py#L756)）通过 `"action": "edit" if images else "generate"` 明确告诉上游是编辑操作。

## 拟议变更

### 变更：在 `build_image_prompt` 中添加图生图编辑指示

**文件**：[services/protocol/conversation.py](file:///d:/GithubProject/chatgpt2api/services/protocol/conversation.py)

#### 修改点 1：扩展 `build_image_prompt` 函数（第 216-222 行）

**当前实现**：
```python
def build_image_prompt(prompt: str, size: str | None, quality: str = "auto") -> str:
    hints = []
    if size:
        hints.append(f"输出图片尺寸为 {size}。")
    if quality:
        hints.append(f"输出图片质量为 {quality}。")
    return f"{prompt.strip()}\n\n{''.join(hints)}" if hints else prompt
```

**修改后**：
- 增加参数 `has_images: bool = False`
- 当 `has_images=True` 时，在 hints 中添加明确的编辑指示，告诉上游模型这是图生图操作，必须基于参考图编辑
- 编辑指示放在 hints 最前，确保优先级

```python
def build_image_prompt(prompt: str, size: str | None, quality: str = "auto", has_images: bool = False) -> str:
    hints = []
    if has_images:
        hints.append("请基于上传的参考图进行编辑修改，保留参考图的主要构图、人物和风格特征，仅根据提示词调整指定部分。")
    if size:
        hints.append(f"输出图片尺寸为 {size}。")
    if quality:
        hints.append(f"输出图片质量为 {quality}。")
    return f"{prompt.strip()}\n\n{''.join(hints)}" if hints else prompt
```

#### 修改点 2：更新调用方（第 665 行 `conversation_events`）

**当前调用**：
```python
final_prompt = prompt_with_global_system(build_image_prompt(prompt, size, quality)) if image_model else prompt
```

**修改后**：传入 `has_images=bool(images)`，让 `build_image_prompt` 知道是否有参考图：
```python
final_prompt = prompt_with_global_system(build_image_prompt(prompt, size, quality, has_images=bool(images))) if image_model else prompt
```

**调用点确认**：通过 Grep 确认 `build_image_prompt` 全项目仅有此一处调用，无其他需同步更新的位置。

## 为什么这样修复

1. **非 codex 模型没有显式的 edit/generate 参数**：codex 模型通过 `image_generation` tool 的 `action` 参数明确区分编辑/生成；非 codex 模型只能通过 prompt 文本隐含指示。
2. **当前 prompt 完全没有编辑指示**：上游 ChatGPT 收到 multimodal 输入后，仅看到用户的原始描述 + 尺寸/质量提示，没有任何"基于参考图编辑"的指示，导致模型可能将图生图当作文生图处理。
3. **添加明确编辑指示后**：上游模型会明确知道这是图生图操作，应当保留参考图的主要内容，仅根据 prompt 调整指定部分。

---

## 影响范围

- **仅影响非 codex 模型**（gpt-image-2 及其前缀变体）：codex 模型走 `stream_codex_image_outputs` 路径，不经过 `conversation_events`，不受影响。
- **仅影响图生图**（有 images 时）：`has_images=False` 时行为完全不变，文生图不受影响。
- **OpenAI 兼容端点 `/v1/images/edits`**：也走 `conversation_events`，会同步受益。
- **任务端点 `/api/image-tasks/edits`**：前端主路径，同样受益。

## 假设与决策

### 假设
1. 用户使用 `gpt-image-2`（非 codex）模型（已通过 AskUserQuestion 确认）
2. "缺少图片参考"指任务成功但生成图片与参考图无关（已通过 AskUserQuestion 确认）
3. 编辑指示文本能被上游 ChatGPT 模型正确理解并遵守

### 决策
1. **仅修改 `build_image_prompt` 函数和其单一调用点**：最小化改动范围
2. **不修改前端**：前端参考图传递逻辑正确，问题在 prompt 构造
3. **不修改上游 API 调用**：`_upload_image` 和 `_start_image_generation` 实现正确
4. **不修改 codex 路径**：codex 模型的 `action: edit/generate` 参数已正确区分编辑/生成

### 不在本次范围
- 前端 File 对象共享的防御性修复
- 非 codex 模型尺寸不一致的缓解（size 提示词强化）
- 前端为非 codex 模型添加尺寸支持提示
- 后端任务编排与账号池调度
